LinkedIn GitHub

Documentation

A simple but efficient Rest API for users to create personal notes

Introduction

This API allows you to make CRUD operations on Users and Notes, It uses MongoDB for the database and its completly done in NodeJS using Express library, it also have user authentication and Json web token for security.
The Database its been populated with some data so you can test it by yourself.

You can download the repo on Github, I have posted it as public and you can find it here

API Endpoints

Click on any endpoint to get more details

Base URL: https://notes-rest-api-v1.herokuapp.com/api

USERS

POST - Add a new user: /users

PUT - Update a user: /users/:id

DEL - Delete a user: /users/:id

POST - Login a user: /auth/login

NOTES

GET - Get all notes from a user: /notes/

GET - Get a note from its ID: /notes/:id

POST - Add a new note: /notes

PUT - Update a note: /notes/:id

DEL - Delete a note: /notes/:id

SEARCH - Admin Section

GET - Get all users: /search/users

GET - Get a user by its ID: /search/users/:id

GET - Search users by their Name or Mail: /search/users/:params

GET - Get all notes: /search/notes

GET - Get a note by its ID: /search/notes/:id

GET - Search notes by their Title, Description or User: /search/notes/:params

POST - Add a new user: /users

Add a new user. Only admins can add other admins

BODY:

(Content-Type application JSON)

key value REQUIRED
name string (min:3 max:24) yes
email string (has to be email format) yes
password string (min:3 max:24) yes
img string no
admin boolean no
HEADERS:

Only admins need this, since you have to be logged as admin to add a new admin

key value REQUIRED
token string (valid JWT token) no (yes only if you want another admin)
EXAMPLE:

Adding a new admin. For this we need two important things, set "admin" attribute to true and send our JWT on the headers section. To get a new JWT you can go to LOGIN USER if you dont have one.

Note: if you try to register an admin without this two parameters the API will register as a normal user.

request:
https://notes-rest-api-v1.herokuapp.com/api/users
body:
{ "name":"admin2",
"email":"admin2@fakemail.com",
"password":"admin2pass",
"admin": true }
headers:
"token":"eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJfaWQiOiI2MjNkMzBmOTMzMTY1YzgzMzZjMGYwODEiLCJpYXQiOjE2NDgyNDQ2NjgsImV4cCI6MTY0ODI1OTA2OH0.yWXfapipl8UMSBj1qjoU6IyW1gpG_3v8kDKgTpDhwKk"

PUT - Update a user: /users/:id

Logged in users can only update their own information. Only admins can update other users and only admins can change normal users to admins.
Mail can not be updated and only sent params will be updated.

PARAMS:

:id has to be a valid MongoID.

key value REQUIRED
:id string (valid MongoID) yes
BODY:

(Content-Type application JSON)

key value REQUIRED
name string (min:3 max:24) no
password string (min:3 max:24) no
img string no
admin boolean no
HEADERS:

If you are an admin you need to send a valid JWT logged as admin to change other users.

key value REQUIRED
token string (valid JWT token) yes
EXAMPLE:

Updating the name of a logged user
To get a new JWT you can go to LOGIN USER if you dont have one.

Note: if you try to change a user to an admin without sending a JWT logged as admin the API will send a bad request.

request:
https://notes-rest-api-v1.herokuapp.com/api/users/623d30f933165c8336c0f081
body:
{ "name":"updated user name"}
headers:
"token":"eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJfaWQiOiI2MjNkMzBmOTMzMTY1YzgzMzZjMGYwODEiLCJpYXQiOjE2NDgyNDQ2NjgsImV4cCI6MTY0ODI1OTA2OH0.yWXfapipl8UMSBj1qjoU6IyW1gpG_3v8kDKgTpDhwKk"
